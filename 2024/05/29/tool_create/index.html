文件操作

您可以使用 fopen( ) 函数来创建一个新的文件或者打开一个已有的文件，这个调用会初始化类型 FILE 的一个对象，类型 FILE 包含了所有用来控制流的必要的信息。下面是这个函数调用的原型：
FILE *fopen( const char *filename, const char *mode );

基本函数

  • int fclose( FILE *fp ); 关闭文件

  • int fputc( int c, FILE *fp ); 写入文件

  • int fputs( const char *s, FILE *fp ); 写入文件，字符串

  • char *fgets( char *buf, int n, FILE *fp );
    函数 fgets() 从 fp 所指向的输入流中读取 n - 1 个字符。它会把读取的字符串复制到缓冲区 buf，并在最后追加一个 null 字符来终止字符串。
    读取文件
    如果这个函数在读取最后一个字符之前就遇到一个换行符 ‘\n’ 或文件的末尾 EOF，则只会返回读取到的字符，包括换行符。也可以使用 int fscanf(FILE *fp, const char *format, …) 函数来从文件中读取字符串，但是在遇到第一个空格和换行符时，它会停止读取。

  • lseek(int fd, off_t offset, int whence);

其中，fd是文件描述符，offset是偏移量，whence是起始位置。whence可以是以下值：
SEEK_SET：文件开头
SEEK_CUR：当前位置
SEEK_END：文件结尾
函数成功时，返回新的文件位置；失败时，返回-1。

函数原型：

  • int open(const char pathname, int flags, … / mode_t mode */ );
    open函数是UNIX和类UNIX操作系统（如Linux和macOS）中的一个系统调用，用于打开或创建一个文件，并返回一个文件描述符。在C语言标准库中，open函数定义在<fcntl.h>头文件中.
    参数说明：
    pathname：这是一个指向以null结尾的字符串的指针，指定了文件的路径。这个路径可以是绝对路径或相对路径。

flags：这是一个整数，用于指定文件打开的模式和行为。这个参数可以是以下一个或多个标志的组合：

O_RDONLY：以只读模式打开文件。
O_WRONLY：以只写模式打开文件。
O_RDWR：以读写模式打开文件。
O_CREAT：如果文件不存在，创建文件。需要与O_WRONLY或O_RDWR结合使用。
O_EXCL：与O_CREAT一起使用，文件必须不存在，否则open调用失败。用于防止覆盖现有文件。
O_TRUNC：如果文件已存在，且以写模式打开，则截断文件的大小为0。
O_APPEND：所有写操作都从文件末尾开始，不影响文件指针。
O_NONBLOCK：以非阻塞模式打开文件（主要用于特殊文件和套接字）。
O_SYNC：所有写操作都应立即写入磁盘。
mode（可选）：这是一个可选参数，当O_CREAT标志被设置时，这个参数才有意义。它定义了新创建文件的权限位。通常使用八进制数来表示，例如0644，其中：

第一个数字通常是0，表示特殊权限位。
第二个数字定义了文件所有者的权限。
第三个数字定义了与文件所有者同组的用户的权限。
第四个数字定义了其他用户的权限。
返回值：
成功：返回一个非负的文件描述符（fd），它是操作系统用来唯一标识打开文件的整数。
失败：返回-1，并设置errno全局变量以指示错误类型。
错误处理：
当open函数失败时，它会返回-1，并且可以通过检查errno来确定错误的原因。常见的错误包括：

EACCES：权限不足。
ENOENT：文件或目录不存在。
EEXIST：文件已存在，且O_CREAT和O_EXCL标志同时被设置。
ENOSPC：没有足够的空间来创建新文件。

  • ssize_t write(int fd, const void *buf, size_t count);

参数说明：
fd：这是一个整数，表示要写入数据的文件的文件描述符。文件描述符是由 open 函数返回的，用于唯一标识一个打开的文件或其他类型的数据流。

buf：这是一个指向数据缓冲区的指针，write 函数将从这个缓冲区中读取数据并写入文件描述符 fd 指定的文件或数据流。缓冲区的数据类型是 void*，意味着它可以指向任何类型的数据。

count：这是一个无符号整数（size_t 类型），表示要写入的字节数。

返回值：
成功：write 函数返回实际写入的字节数。如果返回的值小于 count，则表示遇到了 EOF（文件结束符），或者在写入过程中发生了错误。
失败：返回 -1，并设置 errno 全局变量以指示错误类型。
错误处理：
当 write 函数失败时，它会返回 -1，并且可以通过检查 errno 来确定错误原因。常见的错误包括：

EBADF：文件描述符 fd 不是一个有效的文件描述符，或者指向一个挂起的套接字。
EFBIG：试图写入的数据量超过了文件的最大大小。
EINTR：写入操作被中断。
ENOSPC：没有足够的空间来写入数据。
EPIPE：在管道或套接字上写入，但对方已经关闭了连接。
