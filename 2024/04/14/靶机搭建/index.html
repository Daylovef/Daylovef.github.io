docker

docker是一个软件，是一个运行与linux和windows上的软件，用于创建、管理和编排容器；docker平台就是一个软件集装箱化平台，是一个开源的应用容器引擎，让开发者可以打包他们的应用以及依赖包到一个可移植的镜像中，也可以实现虚拟化，并且容器之间不会有任何接口。

一般项目中一个容器还不够，则需要多个容器，难道就一个个下载，运行吗？这样还是麻烦，有没有一种办法，同时启动多个容器呢？那就是通过docker-compose 技术可以做到同时启动多个容器

docker run -m 256m my_image

-v参数详解

-v参数用于指定挂载关系，格式为宿主机目录:容器目录。

简单描述就是将容器的目录链接上宿主机目录

docker保存的镜像的位置

1、linux保存在”/var/lib/docker/“中；
2、windows保存在”C:\ProgramData\DockerDesktop”中；
3、Mac保存在”com.docker.docker/Data/vms/0/“中

docker 镜像 常用命令
docker run 命令

用于从远程仓库下载镜像，并直接运行（一个命令就可以下载，运行）

eg: 下载Nginx1.16版本

这一个是没有指定版本
docker run -d -p 80:80 nginx

因为每个镜像他都是有自己的端口的比如nginx镜像默认是80端口，这个端口跟本机端口是两回事，我们需要用本机的端口进行映射，比如把本机80端口映射到Nginx的80端口；

所以这里的80:80就是指的将物理机的80端口映射到Nginx镜像的80端口

指定版本nginx下载
如果本机的80端口已经被占用了，就假设我们前面已经下载了nginx，那我们这一次下载指定版本就需要换成81端口
docker run -d -p 81:80 nginx:1.16

下载的时候可能出现一个提示，比如
Unable to find image "nginx1.16" locally

是因为docker run命令下载镜像的时候先判断这个镜像在本地存不存在（有没有已经下载好），如果没有则从远程仓库下载，如果本地有，则直接运行本地下载好的镜像

注意：镜像是镜像，容器是容器，不是说镜像运行起来以后身份变了成了容器；而是镜像运行起来以后镜像还在，可以说产生了一个容器，所以说一个镜像可以运行多个独立的容器

比如我们现在再创建一个nginx1.16的容器，但是我们需要换一个端口
docker run -d -p 82:80 nginx:1.16

docker search 命令

用于搜索镜像，有时候想下载镜像，但想看看有有哪些镜像，就可以用这个命令去查看

下载的时候就可以下载自己想下载的镜像

docker images 命令

查看本地镜像列表（下载了哪些镜像）
有个 IMAGE ID 就是这个镜像的ID值

docker pull 命令

docker run 命令时下载直接运行镜像，而我们只想下载，不想运行，就使用这个指令

docker pull 作用把远程镜像拉取到本地

docker pull httpd

docker push 命令

上传镜像，推送镜像，推到官方仓库

docker rmi 命令

已经运行的镜像不能删除
docker rmi nginx:1.16

删除镜像可以用名字，也可以用他的IMAGE ID

可以同时删除多个镜像：格式就是如下，通过空格隔开可以删除多个容器
docker rmi nginx:latest nginx:1.16

docker save

导出镜像，如果我们想要把镜像导出，放到另一个主机里导入然后运行，比如：
docker save nginx:1.16 -o docker.nginx1.16.tar.gz

docker清除镜像缓存

在一些情况下，由于镜像缓存的原因，Docker镜像拉取失败。可以尝试清除Docker镜像缓存，从而解决拉取失败问题。可以通过以下命令来清除Docker镜像缓存：
docker system prune -a
当然，此操作将会清除所有Docker缓存以及未使用的Docker镜像、容器、网络等信息，慎重执行。
